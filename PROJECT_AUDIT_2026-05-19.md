# Project Audit - 2026-05-19

## Scope

This audit covers the current Vite/React + Tauri/Rust application across architecture, lifecycle ownership, performance, memory pressure, frontend/backend communication, and extensibility. The goal is to keep the system safer under long-running desktop sessions and high gateway traffic without changing product behavior.

## Architecture Map

- Frontend: React UI, React Query cache/service layer, generated Specta IPC bindings, desktop event listeners, and focused hooks for live dashboard freshness.
- Backend: Tauri commands plus a Rust gateway runtime that owns proxy routing, provider failover, session binding, circuit breaker state, request logging, OAuth refresh, and local configuration/DB services.
- Communication: Generated IPC commands handle request/response flows; Tauri events publish gateway status, circuit, request, attempt, and notice streams; React Query bridges those events into cache invalidation and dashboard refresh.
- Persistence: SQLite through bounded buffered writers for request logs and circuit breaker state, with write-through fallback for important request logs under backpressure.
